# Git credentials

GitHub credentials are available in environment variables when running in the Aria Chat desktop app:
- `GIT_USERNAME` — GitHub username configured in Settings > General
- `GIT_PASSWORD` — GitHub personal access token configured in Settings > General

IMPORTANT: Never set or override `$env:GIT_USERNAME` or `$env:GIT_PASSWORD` yourself — these are already populated by the desktop app and must not be modified.

ALWAYS use these credentials for git push. Construct the URL in two steps:
1. Get the remote URL and strip the protocol prefix:
   - Bash: `$repo = $(git remote get-url origin | sed 's|^https://||')`
   - PowerShell: `$repo = (git remote get-url origin) -replace '^https://', ''`
2. Push with credentials embedded in the URL:
   - Bash: `git push "https://$env:GIT_USERNAME:$env:GIT_PASSWORD@$repo" HEAD:<remote-branch>`
   - PowerShell: `git push "https://$env:GIT_USERNAME:$env:GIT_PASSWORD@$repo" HEAD:<remote-branch>`

Never run `git push` without the credential-embedded URL — it will fail.
Never use `interactive: true` on git push commands — the credential-embedded URL handles authentication silently; interactive mode will hang waiting for input.
